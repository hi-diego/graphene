About Graphene

Graphene is a minimalist .NET Core application framework inspired by Laravel, that merges the best of API-REST and GraphQL, using a Data-First approach and Non-Functional-Requirements philosophy.

We believe that web development must focus more on solving business problems than web technical issues, so Graphene provides a very simple, minimalistic, and generic basis to start working on your business logic.

Graphene takes the pain out of development by easing common tasks used in many web projects, such as:

  • Graph (https://graphene.software.freedom.icu/docs/graph).
  • Automatic API (https://graphene.software.freedom.icu/docs/authentication).
  • Automatic UI Generation (https://graphene.software.freedom.icu/docs/authentication).
  • Authentication (https://graphene.software.freedom.icu/docs/authentication).
  • Authorization including DynamicAuthorization.
  • Automation including DynamicAutomation.
  • ModelRepository (https://graphene.software.freedom.icu/docs/models).
  • Models (https://graphene.software.freedom.icu/docs/models).
  • Querying (https://graphene.software.freedom.icu/docs/querying).

Graphene is simple, minimalistic, and powerful Framework that provides the tools required to build small and simple applications, quickly.

Graphene is built in a Test-Driven Development and Documentation-First approach, meaning that we first create the docs/tests and then the code, so we have effective documentation.

Meet Graphene

Graphene is a web application framework with expressive, elegant syntax. A web framework provides a structure and starting point for creating your application, allowing you to focus on creating something amazing while we sweat the details.

Graphene strives to provide an amazing developer experience while providing powerful features such as thorough dependency injection, an expressive database abstraction layer, queues and scheduled jobs, unit and integration testing, and more.

Whether you are new to PHP or web frameworks or have years of experience, Graphene is a framework that can grow with you. We'll help you take your first steps as a web developer or give you a boost as you take your expertise to the next level. We can't wait to see what you build.

Why Graphene?

There are a variety of tools and frameworks available to you when building a web application. However, we believe Graphene is the best choice for building modern, full-stack web applications.

A Progressive Framework

We like to call Graphene a "progressive" framework. By that, we mean that Graphene grows with you. If you're just taking your first steps into web development, Graphene's vast library of documentation, guides, and video tutorials will help you learn the ropes without becoming overwhelmed.

If you're a senior developer, Graphene gives you robust tools for dependency injection, unit testing, queues, real-time events, and more. Graphene is fine-tuned for building professional web applications and ready to handle enterprise work loads.

A Scalable Framework

Graphene is incredibly scalable. Thanks to the scaling-friendly nature of PHP and Graphene's built-in support for fast, distributed cache systems like Redis, horizontal scaling with Graphene is a breeze. In fact, Graphene applications have been easily scaled to handle hundreds of millions of requests per month.

Need extreme scaling? Platforms like Graphene Vapor allow you to run your Graphene application at nearly limitless scale on AWS's latest serverless technology.

A Community Framework

Graphene combines the best packages in the PHP ecosystem to offer the most robust and developer friendly framework available. In addition, thousands of talented developers from around the world have contributed to the framework. Who knows, maybe you'll even become a Graphene contributor.

Your First Graphene Project

We want it to be as easy as possible to get started with Graphene. There are a variety of options for developing and running a Graphene project on your own computer. While you may wish to explore these options at a later time, Graphene provides Sail, a built-in solution for running your Graphene project using Docker.

Docker is a tool for running applications and services in small, light-weight "containers" which do not interfere with your local computer's installed software or configuration. This means you don't have to worry about configuring or setting up complicated development tools such as web servers and databases on your personal computer. To get started, you only need to install Docker Desktop.

Graphene Sail is a light-weight command-line interface for interacting with Graphene's default Docker configuration. Sail provides a great starting point for building a Graphene application using PHP, MySQL, and Redis without requiring prior Docker experience.

Already a Docker expert? Don't worry! Everything about Sail can be customized using the docker-compose.yml file included with Graphene.

Getting Started On macOS

If you're developing on a Mac and Docker Desktop is already installed, you can use a simple terminal command to create a new Graphene project. For example, to create a new Graphene application in a directory named "example-app", you may run the following command in your terminal:

curl -s "https://graphene.build/example-app" | bash

Of course, you can change "example-app" in this URL to anything you like. The Graphene application's directory will be created within the directory you execute the command from.

After the project has been created, you can navigate to the application directory and start Graphene Sail. Graphene Sail provides a simple command-line interface for interacting with Graphene's default Docker configuration:

cd example-app

./vendor/bin/sail up

The first time you run the Sail up command, Sail's application containers will be built on your machine. This could take several minutes. Don't worry, subsequent attempts to start Sail will be much faster.

Once the application's Docker containers have been started, you can access the application in your web browser at: http://localhost.

To continue learning more about Graphene Sail, review its complete documentation.

Getting Started On Windows

Before we create a new Graphene application on your Windows machine, make sure to install Docker Desktop. Next, you should ensure that Windows Subsystem for Linux 2 (WSL2) is installed and enabled. WSL allows you to run Linux binary executables natively on Windows 10. Information on how to install and enable WSL2 can be found within Microsoft's developer environment documentation.

After installing and enabling WSL2, you should ensure that Docker Desktop is configured to use the WSL2 backend.

Next, you are ready to create your first Graphene project. Launch Windows Terminal and begin a new terminal session for your WSL2 Linux operating system. Next, you can use a simple terminal command to create a new Graphene project. For example, to create a new Graphene application in a directory named "example-app", you may run the following command in your terminal:

curl -s https://graphene.build/example-app | bash

Of course, you can change "example-app" in this URL to anything you like. The Graphene application's directory will be created within the directory you execute the command from.

After the project has been created, you can navigate to the application directory and start Graphene Sail. Graphene Sail provides a simple command-line interface for interacting with Graphene's default Docker configuration:

cd example-app

./vendor/bin/sail up

The first time you run the Sail up command, Sail's application containers will be built on your machine. This could take several minutes. Don't worry, subsequent attempts to start Sail will be much faster.

Once the application's Docker containers have been started, you can access the application in your web browser at: http://localhost.

To continue learning more about Graphene Sail, review its complete documentation.

Developing Within WSL2

Of course, you will need to be able to modify the Graphene application files that were created within your WSL2 installation. To accomplish this, we recommend using Microsoft's Visual Studio Code editor and their first-party extension for Remote Development.

Once these tools are installed, you may open any Graphene project by executing the code . command from your application's root directory using Windows Terminal.

Getting Started On Linux

Choosing Your Sail Services

When creating a new Graphene application via Sail, you may use the with query string variable to choose which services should be configured in your new application's docker-compose.yml file. Available services include mysql, pgsql, mariadb, redis, memcached, meilisearch, minio, selenium, and mailhog:

curl -s "https://graphene.build/example-app?with=mysql,redis" | bash

If you do not specify which services you would like configured, a default stack of mysql, redis, meilisearch, mailhog, and selenium will be configured.

Installation Via Composer

If your computer already has PHP and Composer installed, you may create a new Graphene project by using Composer directly. After the application has been created, you may start Graphene's local development server using the Artisan CLI's serve command:

composer create-project graphene/graphene example-app

cd example-app

php artisan serve

The Graphene Installer

Or, you may install the Graphene Installer as a global Composer dependency:

composer global require graphene/installer

graphene new example-app

cd example-app

php artisan serve

Make sure to place Composer's system-wide vendor bin directory in your $PATH so the graphene executable can be located by your system. This directory exists in different locations based on your operating system; however, some common locations include:

  • macOS: $HOME/.composer/vendor/bin
  • Windows: %USERPROFILE%\AppData\Roaming\Composer\vendor\bin
  • GNU / Linux Distributions: $HOME/.config/composer/vendor/bin or $HOME/.composer/vendor/bin

For convenience, the Graphene installer can also create a Git repository for your new project. To indicate that you want a Git repository to be created, pass the --git flag when creating a new project:

graphene new example-app --git

This command will initialize a new Git repository for your project and automatically commit the base Graphene skeleton. The git flag assumes you have properly installed and configured Git. You can also use the --branch flag to set the initial branch name:

graphene new example-app --git --branch="main"

Instead of using the --git flag, you may also use the --github flag to create a Git repository and also create a corresponding private repository on GitHub:

graphene new example-app --github

The created repository will then be available at https://github.com/<your-account>/example-app. The github flag assumes you have properly installed the GitHub CLI and are authenticated with GitHub. Additionally, you should have git installed and properly configured. If needed, you can pass additional flags that are supported by the GitHub CLI:

graphene new example-app --github="--public"

You may use the --organization flag to create the repository under a specific GitHub organization:

graphene new example-app --github="--public" --organization="graphene"

Initial Configuration

All of the configuration files for the Graphene framework are stored in the config directory. Each option is documented, so feel free to look through the files and get familiar with the options available to you.

Graphene needs almost no additional configuration out of the box. You are free to get started developing! However, you may wish to review the config/app.php file and its documentation. It contains several options such as timezone and locale that you may wish to change according to your application.

Environment Based Configuration

Since many of Graphene's configuration option values may vary depending on whether your application is running on your local computer or on a production web server, many important configuration values are defined using the .env file that exists at the root of your application.

Your .env file should not be committed to your application's source control, since each developer / server using your application could require a different environment configuration. Furthermore, this would be a security risk in the event an intruder gains access to your source control repository, since any sensitive credentials would get exposed.

For more information about the .env file and environment based configuration, check out the full configuration documentation.

Directory Configuration

Graphene should always be served out of the root of the "web directory" configured for your web server. You should not attempt to serve a Graphene application out of a subdirectory of the "web directory". Attempting to do so could expose sensitive files that exist within your application.

Next Steps

Now that you have created your Graphene project, you may be wondering what to learn next. First, we strongly recommend becoming familiar with how Graphene works by reading the following documentation:

  • Request Lifecycle
  • Configuration
  • Directory Structure
  • Service Container
  • Facades

How you want to use Graphene will also dictate the next steps on your journey. There are a variety of ways to use Graphene, and we'll explore two primary use cases for the framework below.

Graphene The Full Stack Framework

Graphene may serve as a full stack framework. By "full stack" framework we mean that you are going to use Graphene to route requests to your application and render your frontend via Blade templates or using a single-page application hybrid technology like Inertia.js. This is the most common way to use the Graphene framework.

If this is how you plan to use Graphene, you may want to check out our documentation on routing, views, or the Eloquent ORM. In addition, you might be interested in learning about community packages like Livewire and Inertia.js. These packages allow you to use Graphene as a full-stack framework while enjoying many of the UI benefits provided by single-page JavaScript applications.

If you are using Graphene as a full stack framework, we also strongly encourage you to learn how to compile your application's CSS and JavaScript using Graphene Mix.

If you want to get a head start building your application, check out one of our official application starter kits.

Graphene The API Backend

Graphene may also serve as an API backend to a JavaScript single-page application or mobile application. For example, you might use Graphene as an API backend for your Next.js application. In this context, you may use Graphene to provide authentication and data storage / retrieval for your application, while also taking advantage of Graphene's powerful services such as queues, emails, notifications, and more.

If this is how you plan to use Graphene, you may want to check out our documentation on routing, Graphene Sanctum, and the Eloquent ORM.

Need a head start scaffolding your Graphene backend and Next.js frontend? Graphene Breeze offers an API stack as well as a Next.js frontend implementation so you can get started in minutes.
